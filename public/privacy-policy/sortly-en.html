3. Usage Analytics

Product-improvement analytics is off by default. It is transmitted from Microsoft Store installations only after you explicitly enable App Settings → Usage analytics (optional), and you can turn it off again at any time. Turning it off stops future analytics events. You may request deletion of previously collected records using the contact information below.

  • Collected: random install and session IDs, event name, app version, operating system, language, plan, Store-build status, feature outcome, error type, count ranges, and interactions with update or purchase controls
  • Not collected: file names, folder paths, file or document contents, names, email addresses, or payment method information
  • Purpose: understand active installations and feature trends, diagnose errors, and improve product and purchase flows
  • Retention: normally up to 24 months from the event date, after which records are deleted or converted into aggregate data that no longer distinguishes a person or device

The random install ID contains no name or email address, but it is a persistent identifier that links events from the same installation. For that reason, this policy does not describe it as fully anonymous information.

4. Microsoft Store Subscription Status Sync

For paid-feature access, license verification, and subscription support, active Store subscription details may be sent to the operator's Supabase-based administration store. This processing is separate from the analytics preference and applies only to active paid subscriptions.

  • Information: app-specific random publisher user ID, app version, Microsoft-verified Store subscription, product and SKU identifiers, purchase market, active/renewal/trial status, and start, expiration, cancellation, and last-modified timestamps
  • Not collected: Microsoft account email, user name, card number, or billing address
  • Retention: up to five years from the latest renewal or modification for subscription support and dispute handling, after which records are automatically deleted. Records subject to a separate statutory duty may be isolated for the required period.

5. Optional Cloud AI

Cloud AI is off by default. The following information is transmitted over HTTPS only after you review the notice and choose to start the feature.

  • AI file classification: file name, folders configured as classification choices, and an initial text snippet extracted from a supported document
  • Natural-language rules: the instruction you enter and target file name, extension, size, and age
  • Images and music: original image and music content is not sent for AI document analysis; local metadata is used instead

Requests pass through Sortly's Cloudflare Worker to the OpenAI API. The Sortly operator has not implemented storage of request bodies in its own database. OpenAI API data is not used for model training by default, but abuse-monitoring logs may contain customer content and may be retained for up to 30 days. Exceptions may apply for legal requirements or account-specific data controls.

You may refuse or stop using Cloud AI. Local organization remains available, but cloud classification and natural-language rule features may be unavailable or limited to local processing.

7. Deletion and Your Rights

  • You can delete or change rules, filters, excluded folders, move history, and related settings in the app.
  • Depending on Windows settings, some local app data may remain after uninstall. It can be removed using Windows app-data management tools.
  • To request access, correction, deletion, or restriction of server-side analytics or subscription-sync records, email the address below. We may ask for part of an install ID or subscription identifier to locate the correct record.
  • Information subject to a legal retention duty may be isolated and retained for that period before deletion.

8. Security Measures

External communications use HTTPS. Server storage is protected through restricted access, administrator-only views, database row-level security, and separation of secret credentials. No method of Internet transmission or electronic storage can be guaranteed to be completely secure.

9. Children's Privacy

Sortly is not designed for children under 14 and does not knowingly collect a child's personal information without authorization from a parent or legal guardian. Please contact us if you believe this has occurred.
